# 10. Desktop packaging and installers

## Цель

После стабилизации библиотеки, AI-слоя и редактора упаковать проект как локальное desktop-приложение с установщиками для Windows, Linux и macOS.

Desktop-версия заменяет идею Docker Compose как основной способ распространения. Пользователь должен получать обычное приложение с доступом к локальным директориям, системному диалогу выбора папки и управлению ComfyUI.

## Время выполнения

Эту задачу следует выполнять последней. Ранняя упаковка будет постоянно ломаться из-за изменений страниц, настроек, runtime-интеграции, хранения ключей и структуры backend.

Перед началом необходимо проверить актуальные варианты desktop-оболочки и упаковки Python/web-приложения. Конкретная технология не фиксируется этим документом заранее.

## Целевые платформы

Нужны отдельные результаты для:

- Windows;
- Linux;
- macOS.

Все платформы должны использовать общую бизнес-логику и не требовать отдельных реализаций библиотеки, редактора и AI-инструментов.

Платформенные различия допускаются только в ограниченных слоях:

- системные пути и app data;
- выбор директорий;
- хранение секретов;
- запуск процессов;
- формат установщика;
- desktop notifications и tray, если они будут добавлены.

## Состав приложения

Desktop-сборка должна:

- запускать локальный backend автоматически;
- открывать UI в собственном окне;
- хранить базу, настройки и кэш в системной директории приложения;
- получать доступ к выбранным пользователем папкам;
- управлять собственным процессом ComfyUI;
- корректно завершать дочерние процессы;
- не требовать установленного Poetry или ручного запуска команд.

## Секреты и пользовательские данные

API keys следует хранить через системное безопасное хранилище, если выбранный desktop stack это поддерживает.

Обновление или переустановка приложения не должны удалять:

- настройки;
- список источников;
- библиотеку и альбомы;
- пользовательские templates;
- AI profiles.

Кэш и индекс должны оставаться пересоздаваемыми.

## Установщики

Желаемый результат:

- Windows installer;
- Linux-пакет или переносимый формат, выбранный после проверки актуальной практики;
- macOS application bundle и установочный образ или другой стандартный формат.

Подпись приложений, автоматическое обновление и публикация через магазины могут быть отдельными последующими задачами и не должны блокировать первую desktop-версию.

## Критерии готовности

- Проект устанавливается и запускается без Poetry и ручной настройки Python.
- Windows, Linux и macOS используют одну функциональную модель.
- Системный выбор директорий и Unicode-пути работают на каждой платформе.
- Пользовательские данные переживают обновление приложения.
- Managed ComfyUI корректно запускается и завершается из desktop UI.
- Секреты не хранятся открытым текстом там, где доступно системное хранилище.
- Desktop-упаковка не меняет локальную и однопользовательскую природу проекта.
