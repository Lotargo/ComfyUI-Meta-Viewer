# 06. Prompt skills research

## Цель

Создать сильные специализированные skills для реконструкции и написания prompt под основные семейства генеративных моделей.

Skills должны компенсировать ограниченные знания слабых локальных и облачных моделей, включая модели примерно на 2–4B параметров. Их следует рассматривать как текстовый аналог адаптера: исполнитель получает не общие советы, а подробную систему работы, примеры и проверяемый формат результата.

## Обязательное исследование

Запрещено писать финальные skills только по памяти модели или на основе нескольких шаблонных prompt.

Перед созданием каждого skill исполнитель обязан провести отдельное веб-исследование и найти актуальные материалы, примеры и практические рекомендации 2025–2026 годов.

Исследование должно охватывать:

- официальную документацию и model cards, если они доступны;
- сильные community guides и сравнительные эксперименты;
- реальные примеры удачных prompt;
- различия между естественным языком, тегами и гибридным синтаксисом;
- сложные композиции, динамические сцены, взаимодействие нескольких персонажей и непортретные задачи;
- управление ракурсом, позой, светом, окружением, материалами и визуальным стилем;
- использование negative prompt и случаи, когда он не нужен;
- типичные ошибки конкретного семейства моделей.

Источники и выводы исследования нужно сохранить с датой, чтобы skills можно было пересматривать при изменении моделей и практик.

## Базовые skills

На первом этапе необходимы три отдельных профиля:

### Flux-like

Для FLUX и Chroma как основной natural-language группы. Z-Image может использовать близкий способ описания сцены, но считается отдельным compatibility mode с собственным negative-prompt и guidance behavior.

### SDXL

Для моделей семейства SDXL с более традиционной структурой positive и negative prompt, управлением качеством, композицией и визуальными признаками.

### Pony

Для Pony-подобных моделей, где важны score/source/rating controls, порядок признаков, поддержка естественных captions и dataset tags, а также checkpoint-specific conventions.

Нельзя создавать один универсальный prompt и менять только заголовок семейства.

## Требования к структуре skill

Каждый исследовательский skill должен содержать:

- точную область применения;
- пошаговый анализ входного изображения или пользовательского запроса;
- правила построения композиции;
- правила описания персонажей, объектов и взаимодействий;
- работу со светом, камерой, позой и окружением;
- правила positive и negative prompt;
- хорошие и плохие примеры с объяснением различий;
- типичные ошибки слабых моделей;
- жёсткий формат финального ответа;
- самопроверку результата перед выдачей.

Инструкции должны быть явными. Нельзя предполагать, что базовая модель сама знает правильный порядок тегов, особенности Pony или способы описания сложной сцены.

## Проверка качества

Skills необходимо проверить минимум на:

- портрете;
- full-body или single-character сцене;
- динамическом действии;
- сложном ракурсе;
- предметной или интерьерной сцене;
- пейзаже или окружении;
- стилизации;
- реконструкции prompt по изображению без metadata.

Сцена с двумя и более независимыми персонажами используется как capability-test, а не как обязательный успешный сценарий. Для бюджетных моделей корректным результатом проверки может быть статус `limited`, `experimental` или `unsupported`. Хорошая инструкция не должна маскировать фундаментальное ограничение checkpoint.

Результат оценивается не по красоте текста, а по пригодности prompt для выбранного семейства моделей и честности зафиксированных ограничений.

## Результат исследования

Канонические `app/ai/prompting/content/profiles/{flux,sdxl,pony}/base.md` фиксируют исследованную базу семейства. Существующий `load_skill()` остаётся compatibility API и читает те же файлы без отдельной копии prompt knowledge.

Подробные источники, даты, доказанные факты, проектные эвристики и аудит файлов сохранены в `research_notes.md`.

Family bases дополнены отдельными operation manifests, scenario manifests, modifiers и output contracts согласно [06A. Prompt profile and agent execution architecture](06A_PROMPT_PROFILE_AND_AGENT_EXECUTION_ARCHITECTURE.md).

## Расширение

Новые профили, например для Anima, B1 или других семейств, позднее можно подключать как отдельные канонические profiles. Они не должны раздувать базовую логику приложения или создавать отдельную ручную копию для каждого provider/agent host.

Checkpoint-specific возможности должны подтверждаться model metadata и практическими тестами. Число параметров модели может быть ориентиром, но не является единственным capability-критерием.

## Критерии готовности

- Для Flux-like, SDXL и Pony созданы разные исследованные skills.
- У каждого skill сохранены источники и дата исследования.
- Skills понятны слабой модели без внешних знаний о prompt engineering.
- Примеры покрывают портреты, single-character и непортретные задачи.
- Ограничения сложных композиций и нескольких персонажей зафиксированы честными capability-статусами.
- Формат результата стабилен и пригоден для автоматической подстановки в редактор.
- Качество проверено практическими тестами на нескольких моделях, а не только чтением текста.
- Research layer отделён от runtime/compiler architecture задания 06A.
