# 02. Source monitoring and cloud directories

## Цель

Автоматически поддерживать индекс подключённых директорий в актуальном состоянии и одинаково работать с обычными локальными папками и папками, синхронизируемыми Google Drive, OneDrive, Яндекс Диском, Dropbox, Syncthing или другим desktop-клиентом.

Приложение работает только с файловой системой. API облачных провайдеров, OAuth и авторизация аккаунтов не нужны.

## Источник файлов

Каждая подключённая директория рассматривается как отдельный источник. Для источника необходимо хранить путь, имя, состояние, настройку рекурсивного сканирования и признак включения.

Когда источник отключён пользователем:

- наблюдение останавливается;
- его assets скрываются из обычного интерфейса;
- записи индекса и связи с альбомами не удаляются;
- повторное включение запускает сверку директории.

## Наблюдение и сверка

Нужна комбинация двух механизмов:

- файловый watcher для быстрой реакции на создание, изменение, переименование и удаление;
- периодическая полная сверка как источник истины после пропущенных событий, перезапуска приложения или массовой синхронизации.

Не нужно размещать скрипты, маяки и служебные файлы внутри пользовательских директорий.

Перед обработкой нового файла следует убедиться, что копирование завершено: размер и время изменения должны стабилизироваться. Частые события от облачного клиента объединяются через debounce.

## Доступность источника

Основной сигнал — доступность самой директории и конкретных файлов, а не только наличие интернета.

Рекомендуемые состояния:

- `disabled`;
- `available`;
- `partially_available`;
- `unavailable`;
- `reconnecting`;
- `error`.

Если источник временно недоступен, приложение не должно считать все его файлы удалёнными. Нужно показать спокойное уведомление и периодически пытаться восстановить доступ. Отсутствие интернета можно использовать как дополнительное объяснение, но локально доступные файлы должны продолжать работать.

## Критерии готовности

- Новые, изменённые, переименованные и удалённые файлы отражаются без ручного повторного добавления папки.
- Поддерживается опциональное рекурсивное сканирование.
- Массовая синхронизация не создаёт сотни параллельных обработок одного файла.
- Временно отключённый диск или интернет не уничтожает индекс источника.
- Пользователь может отключить и снова включить источник.
- После reconnect выполняется сверка и обработка накопившихся изменений.
