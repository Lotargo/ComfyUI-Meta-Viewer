# 03. Media library, albums and favorites

## Цель

Добавить виртуальную организацию коллекции без перемещения и переименования исходных файлов на диске.

Текущая страница просмотра уже достаточно насыщена, поэтому библиотека должна получить отдельную HTML-страницу, а не ещё одну крупную вкладку внутри существующего интерфейса.

## Разделение понятий

Физическая директория является источником файлов. Альбом, избранное, тег и системная выборка являются виртуальными представлениями поверх проиндексированных assets.

Один asset может:

- находиться в нескольких альбомах;
- быть отмечен как избранный;
- иметь пользовательские теги и заметку;
- оставаться в библиотеке даже при временной недоступности источника.

Добавление в альбом не должно копировать или перемещать файл.

## Страница Library

Новая страница должна поддерживать как минимум:

- список альбомов;
- создание, переименование и удаление альбома;
- добавление и удаление assets из альбома;
- избранное;
- выбор обложки;
- множественный выбор;
- массовые операции;
- фильтрацию и сортировку.

Удаление альбома удаляет только виртуальную коллекцию. Удаление asset из альбома не удаляет его из индекса и с диска.

## Системные коллекции

Некоторые представления можно реализовать как системные фильтры, а не как физические или искусственно созданные папки:

- Favorites;
- Without metadata;
- Recently added;
- Unavailable;
- Images;
- Videos;
- Not rated.

## Идентичность файлов

Связи альбомов не должны без необходимости теряться при переименовании файла. Для этого нужно предусмотреть устойчивый идентификатор или fingerprint содержимого, позволяющий отличить переименование от удаления старого и появления нового asset.

Сложный поиск дубликатов можно оставить на будущее, но модель данных не должна полностью зависеть только от текущего имени файла.

## Критерии готовности

- Библиотека открывается как отдельная страница.
- Альбомы и избранное не меняют физические файлы.
- Один asset может находиться в нескольких альбомах.
- Отключение или временная недоступность источника не разрушает виртуальные связи.
- Пользователь чётко видит разницу между удалением из альбома, удалением из индекса и физическим удалением файла.
- Интерфейс поддерживает удобную работу с сериями изображений через множественный выбор.
